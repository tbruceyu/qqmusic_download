# QQ音乐自动播放功能 - 实现计划

## 功能概述
新增自动顺序播放功能：点击「开始」按钮后，自动顺序点击`.songlist__list`列表中的歌曲，每5秒点击下一首，直到最后一首自动停止。点击「停止」可中断播放。

## 核心功能点
### 1. 界面修改
- 将原有「刷新」按钮改为状态切换按钮
- 未运行时显示「▶️ 开始自动播放」
- 运行中显示「⏸️ 停止自动播放」
- 显示当前播放进度：第N首/共M首

### 2. 自动播放逻辑
- 读取页面中`.songlist__list`下的所有歌曲项
- 按列表顺序，每5秒点击下一首歌曲的播放按钮
- 播放到最后一首时自动停止，按钮重置为「开始」
- 支持手动点击「停止」按钮随时中断播放

### 3. 歌曲列表解析
- 从`.songlist__list li` 提取所有歌曲
- 每首歌曲提取信息：歌曲名、歌手、播放按钮元素
- 点击逻辑：模拟点击播放按钮触发播放

### 4. 状态管理
- 运行状态：运行中/停止
- 当前索引：当前播放到第几首
- 定时器管理：正确清理定时器避免内存泄漏

## 实现步骤
### 第一步：修改popup界面
1. 修改按钮文字和样式，支持开始/停止状态切换
2. 添加进度显示区域
3. 保留原有清空和导出功能不变

### 第二步：content.js实现核心逻辑
1. 实现歌曲列表解析函数，提取所有歌曲的播放按钮
2. 实现自动点击函数，按顺序点击并记录当前索引
3. 实现定时器控制逻辑，每5秒执行一次
4. 实现开始/停止控制接口，接收popup指令
5. 实现状态查询接口，返回当前进度和状态

### 第三步：popup.js交互逻辑
1. 按钮点击事件处理，发送开始/停止指令到content script
2. 实时查询播放进度并更新界面
3. 状态变化时更新按钮文字和样式

### 第四步：异常处理
- 页面没有songlist__list时提示"未找到歌曲列表"
- 列表为空时提示"歌曲列表为空"
- 播放过程中页面跳转或DOM变化时自动停止并提示
- 定时器异常自动清理

## 技术实现要点
### 点击逻辑
- 优先点击歌曲项的`.list_menu__play`播放按钮
- 如果没有找到，点击歌曲名称链接也可触发播放
- 点击时模拟真实用户点击事件

### 定时控制
- 使用`setInterval`实现5秒间隔
- 停止和完成时立即清除定时器
- 页面unload时自动清理定时器

### 通信机制
-  popup → content：发送start/stop/getStatus指令
- content → popup：返回当前状态、进度、错误信息

## 界面设计规范
- 按钮样式区分运行/停止状态：运行时为红色，停止时为绿色
- 进度显示清晰："正在播放：歌曲名 (第N首/共M首)"
- 状态提示明确：运行中/已停止/播放完成/无列表等
